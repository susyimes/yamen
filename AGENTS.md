# AGENTS.md - Yamen Agent Infrastructure

Yamen 是一个以中国古代县衙 / 州府制度为灵感的轻量多 Agent 架构。

## 目标

这不是“大而全”的朝廷系统，而是一个适合日常事务和中小任务的少 Agent 协作模型。

核心要求：
- 默认短链路
- 默认少角色
- 默认快处置
- 风险任务才加复核
- 超复杂任务应升级到更重制度（如 Edict）

## 核心角色

- `menfang`：门房 / 受理
- `xianling`：县令 / 主官决断
- `zhubu`：主簿 / 任务单与文书
- `kuaishou`：快手 / 承办执行
- `dianshi`：典史 / 风控复核
- `zhangfang`：账房 / 台账与数据（可选）

## 基础流程

### 直办
门房 → 县令 / 快手 → 回禀

### 立案
门房 → 县令 → 主簿 → 快手 → 回禀

### 复核案
门房 → 县令 → 主簿 / 快手 → 典史 → 县令回禀

## 边界

- 门房不做复杂规划
- 县令不直接承办细节执行
- 主簿不替代快手长期执行
- 快手不自己给自己复核
- 典史不负责主执行，只负责风险和放行

## 输出原则

所有 Agent 输出优先使用结构化字段：
- 标题
- 摘要
- 风险等级
- 处理模式
- 主办角色
- 当前状态
- 阻塞项
- 回禀摘要

## 升级机制

符合以下任一条件时，建议升级到更重制度：
- 跨多个专业域且依赖复杂
- 高风险部署 / 外部发布 / 高权限操作
- 需要正式审议或多轮会签
- 周期较长且需要强审计回放
